  3. Create an index.html file

Hosting a website with Github Pages

Create a new Github Repository

To create a website with Github Pages you first need to create a new repository. The name of this repository is crucial and must follow a specific pattern. If you have not already, login to your Github account. If you do not have a Github account, you can sign up for one here. Once you have logged in, click on the plus button in the upper right hand corner of your screen and select "New repository" (See Image 1.1).

Image 1.1

You now need to give your new repository a name. The name MUST BE your Github username followed by ".github.io". For example, if your Github username is JohnDoe, then your Github repository must be "JohnDoe.github.io". (See Image 1.2)

Image 1.2

Remember: Mark your new repository as public before selecting the green "Create repository" button.

Clone the Repository

On your local computer open the Terminal Application(or Gitbash on Windows). Navigate to the directory where you would like to store your project ie: Documents folder. Next, in your Web Browser navigate to Github.com, and go to the main page of the repository. Above the list of files select the green "Code" button. (see Image 1.3)

Image 1.3

From the dropdown that opens select "ssh" and copy the repository link by selecting the "clipboard" button. (See Image 1.4)

Image 1.4

Note: You will need to have previously setup your Github ssh keys. If you have not done so follow this guide.

Back in the Terminal Application, type Git clone and then paste the link that you copied earlier. (See Figure 1.1)

$ git clone https://github.com/YOUR-USERNAME/YOUR-REPOSITORY

Figure 1.1

Note: make sure you are in the directory where you would like the project to be stored before using the git clone command.

Press enter to create a local clone of the repository. Once finished you should see an output similar to figure 1.2.

$ git clone https://github.com/YOUR-USERNAME/YOUR-REPOSITORY

> Cloning into `Documents`...

> remote: Counting objects: 10, done.

> remote: Compressing objects: 100% (8/8), done.

> remove: Total 10 (delta 1), reused 10 (delta 1)

> Unpacking objects: 100% (10/10), done.

Figure 1.2

Create your index.html file

Now that your repository has been cloned locally it is time to add your first file. In the Terminal Application navigate into the repositories local directory on your machine by using the "change directory" command cd. (See figure 1.3)

$ cd username.github.io

Figure 1.3

Remember: Replace username with your Github username.

Now that you have set your directory to the local repository we can create our file by typing touch index.html. (See Figure 1.4)

$ touch index.html

Figure 1.4

Remember: Type the command exactly as shown in Figure 1.4 above.

Press enter and the file will have been created in the current directory. Open the file in your editor of choice ie: VS Code, Sublime Text etc... and add some html code to build your page. You are now ready to add, commit and push your index.html file to Github.

Add, Commit, and Push index.html file to Github

Back in the Terminal Application type git status you should see an output similar to Figure 1.5.

$ git status
On branch main
Your branch is up to date with 'orgin/main'

Untracked files:
(Use "git add <file>..." to include in what will be commited
index.html

Figure 1.5

Notice that "index.html" is red, this is because the file has not yet been added to the repository. You do this by using the git add <file>... command. (see Figure 1.6)

$ git add index.html

Figure 1.6

Press enter and the file will be added to the repository. Use use git status again to see what has changed. Notice that "index.html" is now green, this means it is ready to be committed.

To commit the file you will need to use the git commit command. (see Figure 1.7)

Note: The git commit command requires a message, in the figure below replace commit message goes here with a custom message.(The ' ' are required in the commit message)

$ git commit -m 'commit message goes here'

Figure 1.7

Now all that is left is for you to push your commit to Github this is done by simply typing git push. Press enter and the index.html file will be placed in the Github repository. If you open the repository in your web browser and hit refresh you will see in the list of files that "index.html" now appears. You may now enter your portfolio url into your web browser.

Remember: Your portfolio url is the name of your repository which should follow the pattern of username.github.io

Note: If you see a 404 error after entering the url in the web browser try clearing your browsers cache and then refreshing the page after several minutes.

You now have a live Github Pages website, great work!
